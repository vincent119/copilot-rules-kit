# Copilot Pull Request 描述規範

本檔定義 PR 自動生成內容的語氣、結構與語言準則。
所有自動產生的描述，應可**直接貼入 PR 並無須人工修飾**。

---

## 通用原則

- 一律使用 **正體中文** 撰寫。
- 語氣：中性、精確、可審核。
- 不得加入 emoji、表情符號、裝飾性文字。
- 各段開頭需明確描述修改動機與結果，避免空泛用語。
- 生成內容須完整涵蓋：
  - **修改目的**
  - **變更摘要**
  - **影響範圍**
  - **風險與回滾方案（如適用）**

---

## 描述結構

### 🧩 標準格式

```markdown
### ✏️ 修改目的
（簡述為何需要此變更，著重「為何」而非「怎麼做」。）

### 🧱 變更摘要
（列出主要修改內容，建議使用條列式。）

### 📈 影響範圍
（描述受影響的模組、部署、或使用者行為。）

### ⚙️ 測試與驗證
（列出已執行的測試、驗證步驟與通過結果。）

### 🩹 風險與回滾方案
（若修改具潛在風險，說明回滾方式或安全措施。）

### 🔗 關聯議題
（如適用，附上 issue 或 ticket 連結。）
